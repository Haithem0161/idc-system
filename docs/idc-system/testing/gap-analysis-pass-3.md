# Gap Analysis Pass 3 -- Test Plans vs Build Specs

_Date: 2026-05-13_

Pass 3 re-compares every `docs/idc-system/phase-XX.md` build spec against the matching `docs/idc-system/testing/phase-XX-test.md` test plan AFTER the 112 Pass 1 §9.x additions and the 132 Pass 2 §10.x additions landed. A gap is a scenario, command, route, schema rule, sync contract, conflict policy, snapshot, performance SLO, coverage gate, or edge case that the build spec promises but the union of `§1-§6 + §9 + §10` in the test plan still does not verify. New exposure introduced by §9 or §10 rows themselves (a row that names a helper or invariant that itself has no test, or contradicts the build spec) is in scope for Pass 3.

Cross-cutting items legitimately delegated to `security.md`, `sync-conflicts.md`, `i18n-rtl.md`, `performance-soak.md`, or another phase plan (and listed in the owning test plan header's `Out of scope` line) are NOT counted.

Methodology per `.claude/rules/planning.md` Gap Analysis Methodology (Pass 2+) applied to the testing surface defined in `.claude/rules/testing.md` §3-§6.
